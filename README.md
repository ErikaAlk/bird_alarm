# 鸟瘾闹钟

一个用真实鸟鸣叫你起床的 Android 闹钟。到点了不是机械的「滴滴滴」，而是从音库里随机抽一只鸟开始啼叫。内置十种鸟鸣，还能从 [xeno-canto](https://xeno-canto.org) 搜索下载世界各地的野生鸟声、或导入本地音频；下载和导入的鸟声都会和内置鸟鸣一起进入随机抽取池，每天叫醒你的可能都是不同的鸟。

这个版本是按「真拿它当起床闹钟用」反复打磨出来的，重点解决了原版在锁屏 / 息屏下不好用的问题：

- **锁屏不解锁就能关**——响铃时锁屏页直接显示「正在叫的是哪只鸟」加关闭 / 贪睡，不用解锁进 App（已去掉原版的强制认鸟挑战）。
- **不会漏响、不会乱响**——修了一连串真机问题：整夜后到点只亮屏不响、通知和遮罩各放一只鸟「两只叠着响」、在通知里关了 App 内还残留一个遮罩要点两次等。
- **提前知道要响**——响铃前 10 分钟弹倒计时通知，可直接在通知里取消这次；Android 16 上会提级为 Live Update 状态栏胶囊。
- **按中国节假日重复**——自定义星期 / 工作日 / 法定节假日三选一，节假日和调休数据在线实时获取，跨年自动可用。
- **深色模式**——跟随系统，深夜响铃不刺眼。

完整的逐版本改动见下方[更新记录](#更新记录)。**本仓库只维护 Android，iOS 不保证可用。**

> **关于这个 fork**：本仓库是基于原作者 [oastwy/bird_alarm](https://github.com/oastwy/bird_alarm) 按**个人使用习惯**做的修改版。所有改动仅出于自用需要（去掉强制认鸟挑战、锁屏直接关闹钟、节假日重复、深色模式、闹钟 Live Updates 等），**与原作者无关**，原作者不对这些改动负责。

## 功能

- **鸟鸣闹钟**：每次响铃从音库随机抽一只鸟，内置鸟鸣加上你下载 / 导入的鸟声都在抽取范围内
- **锁屏直接关闹钟**：响铃时在锁屏页直接显示「正在叫的是哪只鸟」+ 关闭 / 贪睡 5 分钟，无需解锁进 App（已移除原版的强制认鸟挑战）
- **响铃前 10 分钟倒计时**：提前弹出倒计时通知，可在通知里直接取消这次响铃；Android 16 上会提级为 Live Update 胶囊
- **多种重复方式**：自定义星期、中国工作日、中国法定节假日三选一，节假日 / 调休数据在线实时获取
- **鸟声库管理**：搜索、下载、导入本地音频，支持 IOC 鸟类名录检索
- **深色模式**：跟随系统自动切换，深夜响铃不刺眼
- **多组闹钟**：支持设置多个闹钟并独立管理

## 重要：必须允许后台运行

**闹钟必须保持 App 在后台持续运行才能正常响铃。**

请务必关闭系统对本 App 的后台自动清理：

### Android
1. 进入「设置」→「电池」（或「省电管理」）
2. 找到「鸟瘾闹钟」→ 选择「不限制」或「允许后台活动」
3. 关闭「自动清理后台应用」中对本 App 的限制

不同手机品牌路径略有差异：
- **小米 / Redmi**：设置 → 省电与电池 → 应用省电 → 鸟瘾闹钟 → 无限制
- **华为 / 荣耀**：设置 → 电池 → 启动管理 → 鸟瘾闹钟 → 手动管理，全部开启
- **OPPO / 一加**：设置 → 电池 → 省电模式 → 应用耗电管理 → 鸟瘾闹钟 → 允许
- **vivo**：设置 → 电池 → 后台耗电管理 → 鸟瘾闹钟 → 允许后台运行
- **三星**：设置 → 电池 → 后台使用限制 → 鸟瘾闹钟 → 不受限制

### 锁屏全屏响铃（部分 ROM 需额外授权）

如果响铃时锁屏上只弹出**横幅通知**、没有全屏响铃页，需要再授一个权限：

- **原生 Android / 大部分 ROM**：设置 → 应用 → 鸟瘾闹钟 → 通知 →「全屏通知 / 在锁定屏幕上全屏显示」→ 允许。（App 启动时也会自动引导你到这个页面。）
- **OPPO / 一加 / realme（ColorOS）**：没有上面那个开关，改为开启「**后台弹出界面**」——设置 → 应用管理 → 鸟瘾闹钟 → 权限（或：手机管家 → 权限隐私 → 权限管理 → 鸟瘾闹钟）→「后台弹出界面」→ 允许；同时把「自启动」也设为允许。
- **小米（HyperOS/MIUI）**：设置 → 应用设置 → 应用管理 → 鸟瘾闹钟 → 权限 →「后台弹出界面」「锁屏显示」→ 允许。
- **vivo（OriginOS）**：设置 → 应用与权限 → 权限管理 → 鸟瘾闹钟 →「后台弹出界面 / 悬浮窗」→ 允许。

> 这些是各家 ROM 的私有权限，App 没法替你打开，需要你手动授权一次。授权后锁屏到点就会弹出全屏响铃页（关闭 / 贪睡），不再只是横幅。

### iOS
iOS 系统限制后台应用运行，建议将 App 保留在前台（锁屏前不要完全退出），或在锁屏前确认闹钟已设置成功。

## 使用 xeno-canto 搜索鸟声

在鸟声库中点击搜索，输入鸟种名称（中文、英文或学名均可）即可从 xeno-canto 获取录音列表。如需更高请求额度，可在设置中填入个人 xeno-canto API Key。

## 更新记录

### v1.1.9（2026-06-21）

- **修复**：v1.1.8 让锁屏通知恢复后，真机实测暴露出**两只鸟叠着响**——通知里是一只鸟（原生引擎选的），App 内响铃遮罩却是另一只下载的鸟（Flutter 选的），两路音频同时播；点遮罩里的「关闭」两只都停，点通知里的「关闭」只停原生那只、遮罩里的还在叫，必须进 App 才能彻底关。根因是 Flutter 前台计时器 `_checkAlarms` 在 app 可见时仍会**自己随机选一只鸟、用 Flutter 播放器播**（`_ring(alarm)` 默认 `useNativeAudio=false`），与原生引擎各放各的。现改为：Android 上 `_checkAlarms` 不再自行放音，而是**消费原生这一轮**、用原生选定的同一只鸟显示遮罩（`useNativeAudio=true`），全程只有一路音频、通知与遮罩同鸟。
- **改动**：响铃遮罩每秒轮询原生 `isAlarmRinging`，当铃声是从**通知**的关闭/贪睡键停掉时，遮罩自动关闭——让「通知关闭」与「App 内关闭」行为一致，不再残留一个静默遮罩。

### v1.1.8（2026-06-21）

- **修复**：真机整夜起床闹钟实测——响铃那一刻**没有任何通知、没有全屏响铃页、没有流体云 Live Update**，只是亮屏，必须解锁进 App 才能关。根因是锁屏息屏下按时触发的是 `AlarmManager` 广播：`AlarmReceiver` 抢先自己起播了鸟声并写了 `last_trigger_at`，导致随后 `AlarmSoundService.ring()` 的去重守卫（`now - lastTrigger < 3s && isPlaying()`）在**发响铃通知之前**就 `return` 了——而那条通知是全应用**唯一**带「全屏意图 + 关闭/贪睡键 + Live Update 提级」的通知。现改为：即便判定重复触发，也先补发响铃前台通知再返回，全屏意图/流体云/关闭键随之恢复，并满足前台服务启动契约。
- **修复**：进 App 点「关闭」后会**再弹出一个响铃遮罩、且显示的鸟与第一次不同**，要点两次才能回主页（其实第一次点击铃声就已停）。根因是 `stopAlarmSound` 只清了 `ringing_asset` 没清 `launch_alarm` 标志，加上 `notifyFlutterAlarmSoon` 重复投递 + Flutter 仅靠 `_activeAlarm` 去重（关闭后即解除）——迟到的 `alarmFired`/`resumed` 会再次消费到 `launched=true` 但 `assetPath` 已为空，于是随机选一只鸟弹出第二个遮罩。现改为：原生 `stopAlarmSound` 同步清掉 `launch_alarm`（持久 + 内存态），Flutter 侧再加一个「刚关过闹钟 5 秒内吞掉重复触发」的闩锁。

### v1.1.7（2026-06-21）

- **修复**：下载的鸟鸣不进闹钟抽取池——闹钟"随机选鸟"永远只在 10 个内置鸟鸣里转，下载到本机的鸟鸣从不被播放。根因是原生响铃端（`NativeAlarmPlayer.ensureRingingAsset`）只从内置 `BirdAlarmAssets.sounds` 里 `random()`，且 `scheduleAlarmAt` 把 Flutter 传来的选音直接丢弃。现改为：Flutter 在排闹钟时把**整库里能离线播放的鸟鸣**（内置 asset + 下载到本机的文件）连同「路径→中文名」一起下发给原生，原生在**响铃那一刻**从这个完整音库里随机选——下载的鸟鸣真正进入抽取池，并能在响铃通知里显示正确中文名。`NativeAlarmPlayer.start` 支持按本地文件路径播放下载的鸟鸣。
- **修复**：「当前音库」列表写死只显示前 12 条（10 个内置占满前排），下载/上传的鸟鸣常被挤出看不到。现把下载/自定义条目排到最前且不截断，下载后一定能在音库里看到。

### v1.1.6（2026-06-21）

- **改动**：回到「SDK 36 + 主界面显示在锁屏上」的方案，让**锁屏全屏响铃**与 **Live Updates** 同时成立。`targetSdk` 重新设为 36；恢复 `MainActivity` 的 `showWhenLocked` 与 Flutter **全屏响铃遮罩**——响铃时主界面（含全屏遮罩）显示在锁屏之上，可直接关闭 / 贪睡（无需解锁）。遮罩改为**铺满整屏**（盖住底部导航栏），并自适应深浅色。响铃通知恢复 Live Update 提级（全屏由遮罩负责，二者不冲突）。
- **代价（按使用方确认）**：app 在前台时锁屏点亮会显示本应用界面——这正是让"全屏响铃"和"Live Update"在 Android 14+/16 上同时成立的必要设计。

### v1.1.5（2026-06-21）

- **改动**：`targetSdk` 由 36 降回 **33**，恢复锁屏全屏响铃。真机验证表明：Android 14+（targetSdk≥34）的后台启动限制（BAL）会拦截闹钟从后台拉起全屏页，且全屏通知权限收紧 + ColorOS 流体云会把响铃通知做成胶囊吞掉全屏意图——综合导致锁屏只剩横幅。停在 targetSdk 33 后，闹钟可像旧版一样可靠地从后台直接弹出**全屏响铃页**。重新启用了直接拉起全屏页的逻辑（锁屏/息屏全屏、亮屏解锁仅通知）。
- **取舍**：换来全屏可靠性的代价是 **Live Updates 提级胶囊可能退化为普通通知**（倒计时/响铃通知本身与功能保留）。`compileSdk` 仍为 36。
- 「主界面不常驻锁屏」的行为不变（全屏的是原生响铃页，不是主界面）。

### v1.1.4（2026-06-21）

- **修复**：部分 ROM（ColorOS 等）锁屏响铃只弹横幅、不全屏。日志确认两个原因：(1) `targetSdk 36` 下前台服务/广播直接 `startActivity` 会被系统的后台启动限制（BAL）拦截；(2) 响铃通知一旦被提级为 Live Update，会被 ColorOS 做成"流体云胶囊"，从而吞掉「全屏意图」。修法：**响铃通知不再提级**，全屏响铃页改由通知的「全屏意图」(FSI) 由系统在锁屏/息屏时自动拉起；移除了在 targetSdk 36 上必被拦截的手动 `startActivity`。**「即将响铃」倒计时仍保留 Live Update 胶囊。**

### v1.1.3（2026-06-21）

- **修复**：锁屏响铃不再全屏（升 targetSdk 36 后，Android 14+ 默认不再授予 `USE_FULL_SCREEN_INTENT`，全屏意图被降级成横幅通知）。现在 App 启动会检测并引导授予「全屏通知」权限，授予一次后锁屏即恢复**全屏专用响铃页**。
- **改动**：中国节假日 / 调休日期改为**在线实时获取**（数据源 [timor.tech](https://timor.tech)），按年缓存到本地、每周自动刷新；离线或获取失败时回退到内置 2026 表。不再写死年份，跨年自动可用。

### v1.1.2（2026-06-21）

- **新增**：闹钟重复方式新增「中国法定节假日」选项。设置里改为三选一——自定义星期 / 工作日 / 节假日；选「节假日」只在 2026 放假日响铃（不含调休补班日）。
- **改动**：「鸟瘾闹钟已启用」常驻守护通知改回普通通知，不再提级为 Live Update；只有「即将响铃」倒计时与「正在响铃」才使用 Live Update。
- **改动**：主界面不再常驻锁屏——App 在前台时锁屏点亮不会再显示本应用（响铃时的锁屏界面仍由原生专用响铃页负责）。

### v1.1.1（2026-06-21）

> 真机实测后的修正，均基于个人使用习惯。

- **修复**：响铃通知 / Live Update 里鸟名总显示成「鸟鸣」——原因是通知在选定鸟鸣之前就构建了；改为先确定本轮鸟鸣再建通知。
- **修复**：设置里填写 xeno-canto API Key 时崩溃（`_dependents.isEmpty` 断言）——设置弹窗改为自带 State 的组件，自行管理并释放输入框控制器。
- **修复**：深色模式下首页「报时鸟」卡片里的文字看不清——该卡片恒为浅色，文字 / 图标改用固定深色，不再跟随深色主题变浅。
- **改动**：响铃界面全面交给原生。**锁屏 / 息屏**显示专用的简洁响铃页（大号时间 + 鸟名 + 关闭 / 贪睡，无底栏）；**亮屏解锁**只用通知提醒。移除了原本会连带底部导航栏一起露出的应用内响铃遮罩。
- **修复**：在通知里关闭闹钟后，再进 App 仍停留在响铃界面——应用内响铃遮罩已移除，关闭后进入 App 直接回到主界面。
- **改动**：「鸟瘾闹钟已启用」的常驻守护通知去掉「关闭」按钮（避免误点关掉守护）；「关闭 / 贪睡」只出现在正在响铃的通知上。

### v1.1.0（2026-06-21）

> 以下改动均基于个人使用习惯，与原作者无关。

- **修复**：下载鸟鸣时整个 App 卡死——把原生音频转码从平台主线程移到后台线程执行，下载/转码全程界面保持可滑动、可操作。
- **改动**：移除「强制认鸟挑战」玩法。响铃时锁屏页直接显示「正在叫的是哪只鸟」，并提供「关闭闹钟」与「贪睡 5 分钟」按钮，**无需解锁手机**即可关闭。
- **改动**：亮屏且已解锁时，响铃改用通知（heads-up）提醒，不再强行全屏接管；锁屏 / 息屏时才拉起全屏响铃页唤醒。
- **修复**：`AlarmReceiver` 与 `AlarmSoundService` 共用通知 id 1001 互相覆盖，导致「停止」按钮有时不出现——改为响铃通知统一由前台服务负责。
- **新增**：响铃前 10 分钟弹出倒计时通知，可直接在通知里点「关闭闹钟」取消这次响铃，无需进 App。在 Android 16（API 36）上会提级为 **Live Update**（状态栏胶囊 / 锁屏 / 息屏常驻）；更低版本上显示为普通常驻通知。
- **升级**：`compileSdk` / `targetSdk` 升到 36（Android 16），并声明 `POST_PROMOTED_NOTIFICATIONS` 权限，使上面的通知能被系统提级为 Live Update。**构建前请确保本机已安装 Android SDK Platform 36 且 Android Gradle Plugin ≥ 8.6**，否则构建会报缺少平台。
- **新增**：深色模式，跟随系统自动切换（不提供 App 内手动开关）。浅色保留原有奶油配色；深色下 AppBar、卡片、鸟种/关于面板、前台响铃遮罩、以及锁屏原生响铃页均自动转为深色，深夜响铃不再刺眼。报时鸟插画作为装饰保持原色。
- **修复**：在较新的 Flutter（3.44 / Dart 3.12）下的编译错误——`ThemeData.cardTheme` 改用 `CardThemeData`，并去掉一处误标 `const` 的 `Text`。已用 `flutter analyze` 验证通过。
